# betty-vision — 사진 Vision skill

## 트리거

메시지에 `[Photo: <경로>]` 패턴이 포함되면 이 skill을 적용한다.

## 처리 절차

### 1. 경로 추출

`[Photo: <경로>]` 패턴에서 파일 경로를 추출한다.

예시: `[Photo: /workspace/media/photo_AgACAgIAAxk.jpg] 이게 뭐야?`
→ 경로: `/workspace/media/photo_AgACAgIAAxk.jpg`
→ 캡션: `이게 뭐야?`

### 2. 이미지 읽기

Read 도구로 해당 경로의 이미지 파일을 연다.

```
Read("/workspace/media/photo_xxx.jpg")
```

Read 도구 실패 시 (파일 없음, 권한 오류 등):
- "이미지를 읽을 수 없었어" 응답 후 종료.

### 3. 캡션 유무에 따른 답변

**캡션이 있는 경우**: 캡션의 질문이나 요청에 이미지 내용을 바탕으로 답변한다.

예시:
- "이게 뭐야?" → 이미지에 보이는 내용 설명
- "칼로리 얼마야?" → 이미지의 음식 메뉴 내용을 읽어 답변
- "번역해줘" → 이미지의 텍스트를 번역

**캡션이 없는 경우**: 이미지 내용을 자연스럽게 설명한다.

### 4. 텍스트 포함 이미지 처리

메뉴판, 문서 사진, 스크린샷, 간판 등 텍스트가 포함된 이미지는 OCR 결과를 포함하여 답변한다.

- 텍스트를 정확히 읽어 답변에 포함시킨다.
- 중요한 텍스트 정보(금액, 이름, 날짜 등)는 명확히 언급한다.

## Fallback

1. Read 도구 실패 시: "이미지를 읽을 수 없었어" 응답
2. `[Photo]` (경로 없는 플레이스홀더) 수신 시: "사진을 받았는데 파일에 접근할 수 없었어" 응답
